# @ 提及功能使用指南

## 功能概述

@ 提及功能让你可以像在社交媒体上一样，通过 `@代理名` 的方式指定谁来回应你的消息。这是最自然、最直观的交互方式。

## 基本用法

### 1. @ 单个代理

**输入消息**:
```
@Alice 你觉得这个方案可行吗？
```

**操作**:
1. 在输入框中输入消息
2. 点击 "发送并 @ 代理响应" 按钮
3. Alice 会自动响应

**效果**:
- ✅ 只有 Alice 会响应
- ✅ 其他代理不会被触发
- ✅ 精确控制讨论流程

### 2. @ 多个代理

**输入消息**:
```
@Alice @Bob 你们对这个问题怎么看？
```

**操作**:
1. 在消息中 @ 多个代理
2. 点击 "发送并 @ 代理响应"
3. Alice 和 Bob 会依次响应

**效果**:
- ✅ 被 @ 的代理都会响应
- ✅ 按照 @ 的顺序依次发言
- ✅ 适合需要多方意见的场景

### 3. 按角色名 @

**输入消息**:
```
@产品经理 这个功能的优先级如何？
```

**操作**:
- 可以使用代理的角色名来 @
- 系统会自动识别对应的代理

**支持的格式**:
- `@代理名` - 如 `@Alice`
- `@ 代理名` - 带空格也可以
- `@角色名` - 如 `@产品经理`
- `@ 角色名` - 带空格也可以

## 快速 @ 功能

### 使用标签快速插入

在输入框上方，你会看到所有参与代理的标签：

```
快速 @:  [@Alice]  [@Bob]  [@Carol]
```

**操作**:
1. 点击任意标签
2. `@代理名 ` 会自动插入到输入框
3. 继续输入你的消息

**优点**:
- ✅ 不用手动输入代理名
- ✅ 避免拼写错误
- ✅ 快速高效

## 与其他功能对比

### @ 提及 vs 请求所有代理

**@ 提及**:
- ✅ 精确控制谁发言
- ✅ 避免无关代理响应
- ✅ 讨论更聚焦
- ❌ 需要手动指定

**请求所有代理**:
- ✅ 快速收集所有意见
- ✅ 适合头脑风暴
- ❌ 可能产生无关回复
- ❌ 响应时间较长

**建议**: 
- 明确知道需要谁回答时，使用 @ 提及
- 需要广泛收集意见时，使用请求所有代理

### @ 提及 vs 运行一轮

**@ 提及**:
- ✅ 灵活控制发言顺序
- ✅ 可以跳过某些代理
- ✅ 适合非结构化讨论

**运行一轮**:
- ✅ 所有代理依次发言
- ✅ 结构化讨论
- ❌ 无法跳过代理

**建议**:
- 自由讨论时，使用 @ 提及
- 需要每个人都发言时，使用运行一轮

## 注意事项

### ✅ 正确用法

```
@Alice 你怎么看？                    ✅ 正确
@ Alice 你怎么看？                   ✅ 正确（带空格）
@产品经理 请评估一下                 ✅ 正确（角色名）
@Alice @Bob 你们觉得呢？             ✅ 正确（多个）
```

### ❌ 错误用法

```
@ Alice你怎么看？                    ❌ @ 后没有空格
@alice 你怎么看？                    ❌ 名称大小写错误
@Alic 你怎么看？                     ❌ 名称拼写错误
```

### 💡 提示

1. **使用快速 @ 标签**避免拼写错误
2. **@ 后加空格**让消息更清晰
3. **先 @ 再说话**让代理知道你在跟谁说话
4. **检查代理名称**确保 @ 的是正确的代理

## 常见问题

### Q: @ 了代理但没有响应？

**检查**:
1. 是否点击了 "发送并 @ 代理响应" 按钮
2. 代理名称是否正确（大小写敏感）
3. 查看浏览器控制台是否有错误

**解决**:
- 使用快速 @ 标签避免拼写错误
- 检查会议状态是否为"进行中"

### Q: 可以 @ 不存在的代理吗？

**回答**: 
- 系统只会识别参与会议的代理
- @ 不存在的代理会被忽略
- 建议使用快速 @ 标签

### Q: @ 的顺序重要吗？

**回答**:
- 代理会按照 @ 的顺序依次响应
- 如果 @ 了 3 个代理，会依次等待响应
- 总响应时间 = 单个响应时间 × 代理数量

### Q: 可以在一条消息中多次 @ 同一个代理吗？

**回答**:
- 可以，但代理只会响应一次
- 系统会自动去重
