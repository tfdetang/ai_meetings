# 使用示例

## 场景一：产品功能讨论

### 参与者
- 产品经理 (Alice)
- 软件工程师 (Bob)
- UX 设计师 (Carol)

### 操作流程

1. **创建代理**
   ```
   创建 Alice - 产品经理模板
   创建 Bob - 软件工程师模板
   创建 Carol - UX 设计师模板
   ```

2. **创建会议**
   ```
   主题: 用户登录功能优化
   参与者: Alice, Bob, Carol
   发言顺序: 顺序
   最大轮次: 3
   ```

3. **开始讨论（使用 @ 提及）**

   **你的消息**:
   ```
   我们的用户反馈登录流程太复杂，需要优化。
   目前需要填写用户名、密码、验证码，还要记住我功能不好用。
   @Alice 从产品角度你有什么建议？
   ```
   
   点击 "发送并 @ 代理响应" ✅

   **Alice (产品经理) 的响应**:
   ```
   从产品角度来看，我们需要平衡安全性和用户体验...
   建议：
   1. 支持第三方登录（微信、支付宝）
   2. 优化"记住我"功能
   3. 考虑生物识别登录
   ```

4. **继续讨论（@ 多个代理）**

   **你的消息**:
   ```
   @Bob @Carol Alice 的建议很好，你们从技术和设计角度怎么看？
   ```
   
   点击 "发送并 @ 代理响应" ✅
   
   Bob 和 Carol 会依次响应

   **Bob (工程师) 的响应**:
   ```
   技术实现方面...
   OAuth 2.0 集成、JWT token、安全性考虑...
   ```

   **Carol (设计师) 的响应**:
   ```
   从用户体验角度...
   界面简化、视觉引导、错误提示优化...
   ```

5. **深入讨论（精确 @）**

   **你的消息**:
   ```
   @Carol Bob 提到的技术方案很好，你能否设计一个原型？
   ```
   
   点击 "发送并 @ 代理响应" ✅

6. **导出会议**
   
   点击 "导出 MD" 保存讨论记录

## 场景二：技术方案评审

### 参与者
- 架构师 (David)
- 安全工程师 (Eve)
- DevOps 工程师 (Frank)

### 操作流程

1. **发送背景信息**
   ```
   我们计划将单体应用拆分为微服务架构。
   当前系统：
   - 用户量：100万
   - 日活：10万
   - 数据库：MySQL 单机
   - 部署：单台服务器
   
   请评估这个方案的可行性和风险。
   ```
   
   点击 "发送并请求 AI 响应"

2. **运行多轮讨论**
   
   点击 "🔄 运行一轮" 2-3 次
   
   让所有专家从不同角度分析

3. **针对性提问**
   
   根据讨论内容，向特定专家提问：
   
   ```
   @Eve 关于安全方面，微服务之间的认证授权怎么处理？
   ```
   
   选择 Eve，点击 "请求发言"

## 使用技巧

### 1. 高效的消息发送方式

**@ 提及（最推荐）** ⭐:
```
输入消息: @Alice 你怎么看？
点击 "发送并 @ 代理响应"
```
精确控制谁发言，最自然的交流方式

**@ 多个代理** 🎯:
```
输入消息: @Alice @Bob 你们觉得呢？
点击 "发送并 @ 代理响应"
```
让多个专家同时参与讨论

**请求所有代理** 💬:
```
输入消息 → 点击 "发送并请求所有代理"
```
让所有人都发表意见

**批量讨论** 🔄:
```
点击 "运行一轮"
```
让所有代理依次发言，适合头脑风暴

### 2. 控制讨论节奏

**快速讨论**:
- 连续点击 "运行一轮" 2-3 次
- 适合收集多方观点

**深入讨论**:
- 发送消息后，逐个请求代理发言
- 根据回复内容继续提问
- 适合需要详细分析的场景

**引导讨论**:
- 在消息中 @特定代理
- 提出针对性问题
- 请求该代理发言

### 3. 提问技巧

**好的提问（使用 @）** ✅:
```
@数据分析师 我们的用户留存率下降了 15%，可能的原因是什么？
@产品经理 @技术负责人 请从产品和技术角度分析。
```

**不好的提问** ❌:
```
怎么办？
```

**引导性提问（精确 @）** ✅:
```
@Carol 基于刚才 Bob 提到的技术方案，
你能否从用户体验角度评估一下？
```

**开放式讨论** 💡:
```
这个功能大家怎么看？
```
然后点击 "发送并请求所有代理"

### 4. 会议管理

**短会议** (1-2 轮):
- 快速决策
- 收集初步意见
- 设置最大轮次: 2

**中等会议** (3-5 轮):
- 方案讨论
- 问题分析
- 设置最大轮次: 5

**长会议** (5+ 轮):
- 深入研究
- 复杂问题
- 不设置轮次限制

### 5. 导出和整理

**会议进行中**:
- 随时可以导出当前进度
- 建议每完成一个阶段就导出一次

**会议结束后**:
- 导出 Markdown 格式便于阅读和分享
- 导出 JSON 格式便于数据分析

**整理建议**:
```markdown
# 会议总结

## 关键决策
- 决策 1
- 决策 2

## 待办事项
- [ ] 任务 1 (负责人: Alice)
- [ ] 任务 2 (负责人: Bob)

## 下次讨论
- 话题 1
- 话题 2
```

## 常见使用模式

### 模式 1: @ 提及讨论（推荐）⭐
```
你: @Alice 这个想法怎么样？
Alice 响应
你: @Bob @Carol 你们觉得呢？
Bob 和 Carol 响应
你: @Alice 基于他们的意见，你有补充吗？
Alice 响应
```
适合：自然对话、精确控制

### 模式 2: 顺序讨论
```
你发言 → 运行一轮 → 你发言 → 运行一轮 → ...
```
适合：结构化讨论、方案评审

### 模式 3: 全员讨论
```
你: 这个问题大家怎么看？
点击 "发送并请求所有代理"
所有代理响应
你: 总结和追问
```
适合：收集多方意见、头脑风暴

### 模式 4: 专家咨询
```
你: @专家 A 这个问题怎么解决？
专家 A 响应
你: @专家 A 那如果这样呢？
专家 A 响应
```
适合：深入探讨特定领域问题

## 键盘快捷键

- `Ctrl/Cmd + Enter`: 发送消息（在输入框中）
- `F5`: 刷新页面查看最新消息
- `Esc`: 取消当前操作

---

## 最佳实践

1. **明确会议目标**: 在第一条消息中说明讨论目的
2. **控制参与人数**: 2-4 个代理最佳
3. **适时总结**: 每 2-3 轮总结一次讨论要点
4. **保存记录**: 重要会议及时导出
5. **合理使用**: AI 代理适合辅助思考，不能完全替代人类决策
