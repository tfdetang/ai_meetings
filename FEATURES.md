# AI Agent Meeting System - 功能指南

## 核心功能

### 1. 🔄 自动持续对话

当 AI 代理在回复中 @ 另一个 AI 代理时,被 @ 的 AI 会自动响应,形成连续对话链。

**使用方法:**
1. 在会议室页面开启"🔄 自动持续对话"开关
2. 发送消息时 @ 一个代理
3. 该代理在回复中 @ 另一个代理时,会自动触发响应

**示例:**
```
用户: @产品经理 请评估这个需求
产品经理: 评估完成,@技术负责人 请评估技术可行性
技术负责人: (自动响应) 可行,@设计师 请设计界面
设计师: (自动响应) 设计完成
```

**特性:**
- 支持多层级对话(默认最多 5 层)
- 自动防止无限循环
- 可通过前端开关控制

---

### 2. ⚡ 流式输出

AI 回复实时显示,像打字一样逐字呈现,无需等待完整响应。

**使用方法:**
1. 在会议室页面开启"⚡ 流式输出"开关
2. 请求 AI 响应时,内容会实时显示

**支持的模型:**
- GLM (智谱 AI)
- Anthropic (Claude)
- Google (Gemini)

**特性:**
- 实时显示 AI 回复
- 使用 Server-Sent Events (SSE)
- 自动错误处理

---

### 3. 📝 自动会议纪要

AI 主持人在会议结束时自动生成结构化的会议纪要。

**触发条件:**
- 手动点击"结束会议"按钮
- 达到最大轮次限制

**纪要内容:**
- 总体摘要
- 关键决策
- 待办事项

**使用方法:**
1. 进行会议讨论
2. 点击"结束会议"
3. 系统自动生成纪要并通知

---

### 4. @ 提及功能

通过 `@代理名` 的方式指定谁来回应消息。

**使用方法:**
- 在消息中输入 `@代理名` 或 `@角色名`
- 点击"发送并 @ 代理响应"按钮
- 被 @ 的代理会自动响应

**快速 @ 功能:**
- 点击输入框上方的代理标签
- `@代理名 ` 会自动插入到输入框

**支持格式:**
- `@Alice` - 代理名
- `@ Alice` - 带空格
- `@产品经理` - 角色名
- `@ 产品经理` - 带空格

---

### 5. Markdown 支持

会议消息支持完整的 Markdown 格式渲染。

**支持的语法:**
- 标题 (# ## ###)
- 文本格式 (**粗体** *斜体* ~~删除线~~ `代码`)
- 列表 (有序、无序、任务列表)
- 链接和图片
- 代码块 (带语法高亮)
- 引用
- 表格
- 水平线

**开关控制:**
- 在消息卡片标题栏右侧有"Markdown 渲染"开关
- 可以切换查看原始文本或渲染后的效果

## 使用建议

### 何时启用自动对话?

**✅ 推荐:**
- 多方讨论和头脑风暴
- 技术评审和方案讨论
- 需要多个专家意见的场景

**❌ 不推荐:**
- 简单的单次问答
- 需要精确控制发言顺序

### 何时启用流式输出?

**✅ 推荐:**
- 长文本回复
- 需要实时反馈
- 改善用户体验

**❌ 不推荐:**
- 网络不稳定时
- 需要批量处理时

### 最佳组合

**正式会议:**
```
✅ 自动纪要: 开启
✅ 自动对话: 开启
✅ 流式输出: 开启
```

**快速讨论:**
```
❌ 自动纪要: 关闭
✅ 自动对话: 开启
✅ 流式输出: 开启
```

**一对一咨询:**
```
❌ 自动纪要: 关闭
❌ 自动对话: 关闭
✅ 流式输出: 开启
```

## 故障排除

### 自动对话不工作

**检查:**
1. "自动持续对话"开关是否开启
2. @ 格式是否正确
3. 浏览器控制台是否有错误

### 流式输出不工作

**检查:**
1. "流式输出"开关是否开启
2. 浏览器是否支持 EventSource (SSE)
3. 网络连接是否稳定

### 纪要未自动生成

**检查:**
1. 会议是否有消息内容
2. AI 服务是否可用
3. 查看后端日志
